# 是什么

github 自带的 CI/CD 工具，对应的 gitlab 中也有gitlab-ci。通过其你可以对仓库进行自动化、自定义的构建、测试、部署等操作。也就意味着，当你推送一次代码之后，github 就可以替你完成构建、测试、部署等操作。

# 基本构成

当你的仓库发生某个事件时，你可以设置一个工作流来响应它（或者说被某个事件触发）。如一个推送请求或者一个issue被创建。工作流中可以包含一个或多个作业，这些作业可以按照顺序执行，也可以并行执行。每个作业都运行在它自己的虚拟机中，每个作业会包含一个或多个步骤，你可以在步骤中运行一个脚本，也可以定义或运行另一个可以重复使用的，且可以简化你的工作流程的action
